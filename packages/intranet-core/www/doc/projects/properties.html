Project Properties

A project is defined by its properties (fields) and its associated project components.

Project Fields:

  • Company:
    Is the organization that pays the project expenses.
    There is a predefined company "Internal" that can be used for projects without a paying company.
  • Project Leader:
    The person responsible for the execution and the tecnical success of the project.
  • Project Supervisor:
    The person responsible for the financial success of the project.
  • Project Type:
    Allows to determine the type of services requested by the company. The list of project types can be customized for each company.
  • Project Status:
    Allows to follow-up on the development of a project through several phases of the project lifecycle.
  • Source Language (Project/Translation only): Source language of a translation project
  • Target Languages (Project/Translation only): Target languages (multiples!) of a translation project
  • Subject Area (Project/Translation only): Type of language applied in the translation project
  • Start Date:
    Date of official launch of the project.
  • End Date:
    Deadline for the termination of the project.
  • Budget:
    Estimated total cost of the project
  • Parent Project:
    Allows to create a hierarchy of projects. All costs and revenues of this project are summed up by the parent project.
  • Description:
    Free-text description of the purpose of the project.
  • Note:
    Free-text instructions for all project members.

Project Components:

Project components contain additional information about a project. The available project components depend on the type of the project. Predefined project components include:

  • Filestorage
  • Translation Workflow
  • Project Collaboration
